## 🛠️ 2. Technical Architecture
The framework utilizes a **Multi-Agent Orchestration** pattern:
1. **BA Agent**: Transforms raw business requirements into structured User Stories and Gherkin features.
2. **Dev Agent**: Consumes User Stories to generate clean, modular source code.
3. **QA Agent**: Analyzes code and stories to produce comprehensive test suites (Unit, Integration, E2E).

---

## 🚀 3. Getting Started
1. **Configure API Keys**: Ensure `GEMINI_API_KEY` is set in your environment.
2. **Select Agent**: Navigate to the **Agent Playground**.
3. **Input Context**: Provide requirements or load an example.
4. **Execute**: Run individual agents or the **Full SDLC Chain**.
